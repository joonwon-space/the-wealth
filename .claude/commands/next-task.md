# Next Task

`docs/plan/todo.md`의 첫 번째 미완료 항목(`[ ]`)을 하나 골라 구현하고, 완료 처리 후 커밋한다.

## 실행 순서

### 1. 다음 작업 파악
- `docs/plan/todo.md`를 읽는다
- 첫 번째 `[ ]` 항목을 찾는다
- 어떤 파일을 생성·수정해야 하는지 파악한다

### 2. 구현
- `/plan` 없이 바로 구현한다 (이미 todo가 계획 문서임)
- 해당 작업 하나에만 집중한다 — 다른 항목은 건드리지 않는다
- 프로젝트 규칙을 따른다:
  - TypeScript: 명시적 타입, `any` 금지, immutable 패턴
  - Python: type annotation, PEP 8, `print()` 대신 `logging`
  - 함수 50줄 이하, 파일 800줄 이하

### 3. todo.md 업데이트
- 완료한 항목의 `[ ]`를 `[x]`로 변경한다
- 다른 항목은 수정하지 않는다

### 4. 커밋
아래 형식으로 커밋한다:

```
git add -A
git commit -m "<type>: <작업 내용 요약 (70자 이하)>"
```

커밋 타입:
- `feat`: 새 기능 (모델, API 엔드포인트, 컴포넌트 등)
- `chore`: 설정, 초기화, 인프라
- `refactor`: 코드 구조 개선

커밋 후 완료된 항목과 생성/수정된 파일 목록을 출력한다.
