# E2E Check

핵심 사용자 플로우를 Playwright MCP로 자동 검증하고 성공/실패 리포트를 생성한다.

## 사용법

```
/e2e-check [flow]
```

`flow` 생략 시 전체 플로우를 순서대로 실행한다.

| flow 인자 | 검증 내용 |
|-----------|----------|
| `auth` | 회원가입 → 로그인 → 로그아웃 |
| `portfolio` | 포트폴리오 생성 → 종목 추가 → 거래 기록 |
| `dashboard` | 대시보드 로딩 → 가격 표시 → 알림 배지 |
| `analytics` | 분석 페이지 지표 → 히트맵 → 포트폴리오 추이 |
| `search` | 종목 검색 (국내/해외/초성) → 종목 상세 → 캔들스틱 |

예시:
- `/e2e-check` — 전체 플로우
- `/e2e-check auth` — 인증 플로우만

## 전제조건

- Playwright MCP 활성화 (`.mcp.json` 참조)
- 프론트엔드 dev server: `cd frontend && npm run dev`
- 백엔드 dev server: `cd backend && uvicorn app.main:app --reload`
- 테스트용 계정 (없으면 플로우 내에서 신규 가입)

## 플로우 정의

### Flow 1: auth — 인증

```
1. browser_navigate("http://localhost:3000/register")
2. 이메일/비밀번호 입력 → 가입 버튼 클릭
3. 성공 시 /login 리다이렉트 확인
4. 로그인 정보 입력 → 로그인
5. /dashboard 리다이렉트 확인
6. 설정 페이지 → 로그아웃
7. /login 리다이렉트 확인
```

검증 포인트:
- [ ] 회원가입 성공 후 로그인 페이지 이동
- [ ] 로그인 성공 후 대시보드 이동
- [ ] 잘못된 비밀번호 시 에러 메시지 표시
- [ ] 로그아웃 후 보호 경로 접근 차단

### Flow 2: portfolio — 포트폴리오 관리

```
1. /dashboard/portfolio 이동
2. "포트폴리오 추가" → 이름 입력 → 생성
3. 생성된 포트폴리오 클릭
4. "종목 추가" → 검색창에 "삼성" 입력
5. 검색 결과에서 삼성전자 선택
6. 수량/평균단가 입력 → 저장
7. 거래 탭 → "거래 추가" → 매수 기록 입력
```

검증 포인트:
- [ ] 포트폴리오 목록에 신규 항목 표시
- [ ] 종목 검색 결과 정상 로딩
- [ ] 보유 종목 테이블에 추가된 종목 표시
- [ ] 거래 기록 저장 후 목록 업데이트

### Flow 3: dashboard — 대시보드

```
1. /dashboard 이동
2. 요약 카드 로딩 완료 확인
3. 보유 종목 테이블 렌더링 확인
4. 전일 대비 배지 표시 확인
5. 52주 범위 프로그레스 바 표시 확인
6. 모바일(375px)에서 카드 뷰 전환 확인
```

검증 포인트:
- [ ] 총 평가금액 숫자 형식 (₩ 단위 콤마)
- [ ] 상승/하락 색상 컨벤션 (빨강/파랑)
- [ ] 모바일에서 하단 네비게이션 바 표시
- [ ] SSE 연결 시 가격 업데이트 인디케이터

### Flow 4: analytics — 분석 페이지

```
1. /dashboard/analytics 이동
2. 성과 지표 카드 (총 수익률, CAGR, MDD, 샤프) 로딩 확인
3. 포트폴리오 가치 추이 차트 렌더링 확인
4. 기간 선택 버튼 (1M/3M/6M/1Y/ALL) 동작 확인
5. 월별 수익률 히트맵 렌더링 확인
```

검증 포인트:
- [ ] 스켈레톤 → 실제 데이터 전환
- [ ] 차트 툴팁 표시
- [ ] 기간 변경 시 차트 업데이트

### Flow 5: search — 종목 검색

```
1. Cmd+K 단축키로 검색 다이얼로그 열기
2. "삼성전자" 검색 → 결과 목록 확인
3. 초성 "ㅅ" 검색 → 초성 검색 동작 확인
4. "AAPL" 검색 → 해외 주식 결과 확인
5. 검색 결과 클릭 → 종목 상세 페이지 이동
6. 캔들스틱 차트 렌더링 확인
7. 기간 선택 (1M/3M/6M/1Y) 동작 확인
```

검증 포인트:
- [ ] 검색 다이얼로그 열기/닫기 (Esc)
- [ ] 실시간 검색 결과 업데이트
- [ ] 종목 상세 페이지 펀더멘털 데이터 표시
- [ ] 52주 범위 프로그레스 바

## 오류 처리 및 디버깅

각 단계 실패 시:

1. **스크린샷 캡처** — 현재 화면 상태 저장
2. **콘솔 로그 확인** — `browser_console_messages` 로 JS 에러 확인
3. **네트워크 요청 확인** — API 응답 상태 코드 확인
4. **원인 분류**:
   - 프론트엔드 렌더링 오류 → 컴포넌트 파일 확인
   - API 에러 (4xx/5xx) → 백엔드 로그 확인
   - 인증 오류 → 토큰/쿠키 상태 확인
