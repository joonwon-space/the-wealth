# Fix UI

사용자가 지정한 UI 문제를 스크린샷 → DOM 분석 → 코드 수정 → 재검증 순서로 해결한다.

## 사용법

```
/fix-ui <문제 설명> [페이지 경로]
```

예시:
- `/fix-ui 모바일에서 사이드바가 겹쳐 보여`
- `/fix-ui 다크모드에서 테이블 텍스트가 안 보임 /dashboard`
- `/fix-ui 분석 페이지 히트맵 색상이 반대로 표시됨 /dashboard/analytics`

## 전제조건

- Playwright MCP 활성화 (`.mcp.json` 참조)
- 프론트엔드 dev server 실행 중 (`npm run dev` in `frontend/`)

## 워크플로우

### 1. 현재 상태 캡처

문제가 발생하는 페이지로 이동해 스크린샷을 촬영한다.
페이지 경로가 명시되지 않으면 현재 활성 페이지 사용.

```
browser_navigate → 해당 페이지 이동
browser_screenshot → before 스크린샷 저장
```

문제가 특정 뷰포트에서만 발생하는 경우 해당 뷰포트로 설정 후 촬영:
```
browser_resize(width=375)  → 모바일
browser_resize(width=768)  → 태블릿
browser_resize(width=1280) → 데스크탑
```

### 2. 문제 영역 DOM 분석

`browser_snapshot`으로 페이지 접근성 트리를 확인하고
의심 요소의 CSS 속성과 레이아웃을 파악한다.

확인 항목:
- 요소의 실제 렌더링 크기 및 위치
- 적용된 Tailwind 클래스
- 부모/형제 요소와의 관계
- z-index, overflow 설정

### 3. 소스 파일 식별 및 수정

문제와 관련된 컴포넌트/페이지 파일을 `Read` 도구로 읽는다.
`Edit` 도구로 최소 범위 수정을 적용한다.

수정 원칙:
- 다른 뷰포트/테마에 영향을 주지 않도록 주의
- Tailwind 반응형 접두사 사용 (`md:`, `lg:`, `sm:`)
- 한국 증시 컬러 컨벤션 유지 (상승 `#e31f26`, 하락 `#1a56db`)
- 다크모드 클래스 누락 시 `dark:` 접두사 추가

### 4. 접근성 체크

수정 후 다음 항목을 확인한다:
- [ ] 색상 대비 비율 (WCAG AA 기준: 일반 텍스트 4.5:1, 대형 텍스트 3:1)
- [ ] 인터랙티브 요소에 aria-label 또는 가시적 텍스트
- [ ] 키보드 포커스 이동 순서 논리적
- [ ] 색상 외에 다른 시각적 단서 제공 (아이콘, 텍스트)

### 5. 재검증

Next.js 핫 리로드 후 동일 조건으로 after 스크린샷을 촬영한다.

```
browser_screenshot → after 스크린샷
```

before/after를 나란히 설명하며 수정 내용을 요약한다.

### 6. 회귀 확인

수정 사항이 다른 페이지에 영향을 줄 수 있는 경우 (공통 컴포넌트 변경 등)
연관 페이지도 빠르게 확인한다.

## 출력 형식

```
## Fix UI 결과

**문제**: {문제 설명}
**페이지**: {경로}
**수정 파일**: {파일명}:{라인}

### 변경 내용
{무엇을 어떻게 바꿨는지 1-3문장}

### Before / After
[before 스크린샷 설명]
[after 스크린샷 설명]

✅ 수정 완료 — {뷰포트} × {테마} 에서 검증됨
```
