# Team Implement — Multi-Agent Parallel Implementation

Orchestrate a team of specialist workers to implement tasks from `docs/plan/tasks.md` in parallel, each in an isolated git worktree, then merge and verify all changes.

## Architecture

```
team-implement (orchestrator, you)
  ├── [parallel] backend-worker      → Python backend tasks (worktree)
  ├── [parallel] frontend-worker     → TypeScript frontend tasks (worktree)
  ├── [parallel] infra-worker        → Dependencies, config, security tasks (worktree)
  └── [sequential] implement-synthesizer → Merge worktrees, resolve conflicts, test, commit
```

## Pre-step: Task Classification

1. Read `docs/plan/tasks.md`
2. Collect all incomplete (`[ ]`) items
3. If zero items → read `docs/plan/todo.md`, promote next milestone items to tasks.md, commit
4. Classify each task into a worker bucket by analyzing its file paths:

| Worker | File patterns | Examples |
|--------|--------------|---------|
| **backend-worker** | `backend/**/*.py`, `backend/requirements.txt` | API routes, services, schemas, models, tests |
| **frontend-worker** | `frontend/**/*.ts`, `frontend/**/*.tsx`, `frontend/*.config.*` | Components, pages, hooks, styles |
| **infra-worker** | `*.txt` (requirements), `*.toml`, `.env*`, `docker*`, CI configs, mixed backend+frontend config | Dependencies, security patches, config changes |

5. If a task touches BOTH backend and frontend files → assign to **infra-worker** (it handles cross-cutting concerns)
6. If only one bucket has tasks → launch only that worker (skip empty buckets)

## Pre-step 2: Create working branch

```bash
BRANCH="team-implement/$(date +%Y%m%d-%H%M)"
git checkout -b "$BRANCH"
```

## Phase 1: Launch workers in parallel

Launch ALL applicable workers simultaneously using the Agent tool with `isolation: "worktree"`. Each agent is defined in `.claude/agents/team/implement/`.

For each worker, compose a prompt that includes:
1. The worker's role (from its agent file)
2. The EXACT list of tasks assigned to it (copy the full task text from tasks.md)
3. The branch name to use as base
4. Instructions to follow the worker's agent file for build verification
5. Instructions to commit each task separately with conventional commit messages

**CRITICAL**: Launch all applicable workers in a SINGLE message with multiple Agent tool calls. Do NOT wait between them. Each MUST use `isolation: "worktree"`.

Example prompt for a worker:
```
You are the backend-worker for team-implement. Your job is to implement the following tasks from docs/plan/tasks.md:

Task 1: [full task text]
Task 2: [full task text]

Follow the instructions in .claude/agents/team/implement/backend-worker.md.
Implement each task, verify the build, and commit separately.
```

## Phase 2: Collect results

After all workers complete, each returns:
- A worktree path and branch name (if changes were made)
- A list of completed tasks
- A list of failed tasks (if any)

Read each worker's result carefully.

## Phase 3: Synthesize

Launch the **implement-synthesizer** agent with:
- The current branch name (`$BRANCH`)
- All worktree branch names from Phase 2
- The full task list with worker assignments
- Instructions to follow `.claude/agents/team/implement/implement-synthesizer.md`

The synthesizer will:
1. Cherry-pick or merge each worktree branch into `$BRANCH`
2. Resolve any conflicts
3. Run full build verification (frontend build + backend lint/tests)
4. Update `docs/plan/tasks.md` — mark completed items
5. Update related docs if needed

## Phase 4: Report

After synthesizer completes, output:

```
Team Implement Complete!

Branch: team-implement/YYYYMMDD-HHmm

Workers:
  backend:   {N} tasks completed, {N} failed
  frontend:  {N} tasks completed, {N} failed
  infra:     {N} tasks completed, {N} failed

Total: {completed}/{total} tasks

{If any failed:}
Failed tasks:
- [ ] task description — reason

Next: review changes with `/team-review`, then push with `git push -u origin $BRANCH`
```

## Post-step: PR (if all tasks passed)

If ALL tasks completed successfully:

```bash
git push -u origin "$BRANCH"
gh pr create \
  --base main \
  --head "$BRANCH" \
  --title "team-implement: <summary>" \
  --body "$(cat <<'EOF'
## Tasks completed
- [x] task 1
- [x] task 2
...

## Build verification
All workers passed build verification. Synthesizer confirmed merged result builds clean.

## Workers
- backend-worker: N tasks
- frontend-worker: N tasks
- infra-worker: N tasks

Generated by team-implement agent
EOF
)"
```

## Rules

- ALWAYS launch Phase 1 workers in parallel (single message, multiple Agent calls)
- ALWAYS use `isolation: "worktree"` for each worker — never let workers modify the main working tree
- If a worker has zero tasks, do NOT launch it
- If only 1 worker is needed, still use worktree isolation for safety
- NEVER skip the synthesis phase — unmerged worktrees leave orphan branches
- If a worker fails on a task, it should skip that task and continue with the rest
- Stop conditions: same as auto-task (build fails twice on same task → skip, report)
