| 항목 | 이전 (AR) | 신규 (SIM) |
|------|-----------|-----------|
| 데이터 소스 | transactions + price_snapshots + dividends | 사용자 입력 only |
| 공식 | `eop = bop × (1+r) + flow` | `eop = (bop + flow) × (1 + r)` |

## 기술 결정 사항 (Decision Log)

- **D-1**: `users.simulation_params` JSONB 컬럼 재사용 — 마이그레이션 불필요. 기존
  형태(`SimulationInput` 단일 오브젝트)는 PUT 시 자연 덮어쓰기. 데이터 보존 가치
  낮음 (시뮬용).
- **D-2**: `users.birth_year` 컬럼 **유지** — 향후 다른 메뉴에서 재활용 가능. 컬럼
  drop 마이그레이션 부담 회피.
- **D-3**: 계산은 **클라이언트** 에서. 서버는 저장/조회만. 이유:
  - 셀 편집 시 즉시 피드백 필요 (60+ 행 X 컬럼당 ms 단위)
  - 비즈니스 로직이 단순 (한 줄 곱셈/덧셈), 서버 부담 가치 없음
  - 백엔드는 검증만
- **D-4**: zod 미사용 (기존 패턴) — 단순 수동 validation 으로 충분. 폼 7~8 필드.
- **D-5**: TanStack Table v8 미사용 — 인라인 편집 hook 직접 구현이 더 빠름. 표 작아서
  (60~70 행) 가상화 불요.
- **D-6**: 잔고가 음수가 될 때 **클램프 안 함** — 자산 소진 시점이 그래프에 그대로
  보이는 것이 시각화에 유용.
- **D-7**: CSV import / 다중 시나리오 / 인플레이션은 v2 후보. 우선 시트 1:1 매칭에
  집중.

## 리스크 / 미해결

- 사용자가 메타의 `end_age` 를 늘렸을 때 행을 부분 확장할지 전체 재생성할지 —
  현재 결정: confirm 모달 후 **전체 재생성**.
- 셀 인라인 편집 모바일 UX — 60+ 행 가로 스크롤 + 작은 input 의 터치 정확도.
  TASK-SIM-4 구현 후 모바일에서 한 번 더 점검.
- 한 번 저장된 데이터가 새 스키마와 호환 안 될 때 — GET 응답 파싱 실패 시 메타만
  default 로 폴백, 사용자에게 toast 안내.
