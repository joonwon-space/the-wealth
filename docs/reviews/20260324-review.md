# The Wealth — 프로젝트 리뷰 (2026-03-24)

## 1. 프로젝트 현황 요약

The Wealth는 한국투자증권(KIS) OpenAPI를 활용한 개인 자산관리 대시보드로, 증권사 계좌 연동을 통해 실시간 시세 조회, 포트폴리오 관리, 주식 매매, 투자 분석을 지원하는 풀스택 핀테크 애플리케이션이다.

| 항목 | 현황 |
|------|------|
| 프론트엔드 | Next.js 16 + React 19 + Tailwind v4 + shadcn/ui |
| 백엔드 | FastAPI + SQLAlchemy 2.0 (async) + PostgreSQL 16 |
| 캐시/세션 | Redis 7 (토큰, 가격, 종목 마스터 캐시) |
| 외부 API | 한국투자증권 KIS OpenAPI |
| API 엔드포인트 | 65개 (17개 라우터) |
| DB 테이블 | 11개 |
| 테스트 커버리지 | 백엔드 93% (577+ 테스트) |
| 배포 | Docker Compose, self-hosted (joonwon.dev) |
| CI/CD | GitHub Actions 7개 워크플로우 |
| 모니터링 | Sentry APM + UptimeRobot |

## 3. 구현 완료 기능

### 3.1 인증 및 보안

- 회원가입/로그인 (bcrypt 해싱, JWT access+refresh 토큰)
- Refresh Token Rotation (1회성 JTI, Redis 저장)
- KIS 자격증명 AES-256-GCM 암호화 저장
- SecurityHeaders 미들웨어 (X-Frame-Options, CSP 등)
- slowapi IP 기반 레이트 리밋

### 3.2 포트폴리오 관리

- 포트폴리오 CRUD + 드래그 앤 드롭 순서 변경
- 보유종목 추가/수정/삭제, 수량 및 평균 매입가 관리
- 거래내역 기록 (BUY/SELL) + 거래 메모 인라인 편집
- KIS 계좌 1:1 매핑 및 자동 동기화 (APScheduler 1시간 주기)
- CSV 내보내기 (보유종목 + 거래내역)

### 3.3 주식 매매 (Trading)

- 국내/해외 매수/매도 주문 실행 (KIS API 연동)
- 지정가/시장가 주문, 계좌 유형별 TR_ID 자동 분기 (일반/ISA/연금/IRP)
- Redis 기반 이중 주문 방지 락 (TTL 10초)
- 미체결 주문 조회 및 취소
- 예수금 조회 (국내+해외 합산, Redis 캐시 30초)
- 주문 가능 수량/금액 조회

### 3.4 대시보드 및 시각화

- 포트폴리오 요약 (총 자산, 총 수익/손실, 수익률)
- 보유종목 테이블 (TanStack Table v8, 멀티 컬럼 정렬, 실시간 가격)
- 자산 배분 도넛 차트 (Recharts, 중앙 오버레이)
- 캔들스틱 차트 (lightweight-charts, 일봉 OHLCV)
- 월별 수익률 히트맵
- 포트폴리오 히스토리 차트 (가치 변동 추이)
- 섹터별 배분 분석
- 한국 증시 컬러 컨벤션 (상승=빨강, 하락=파랑)

### 3.5 실시간 데이터

- SSE 기반 30초 주기 가격 업데이트 (KST 09:00~15:30 장중)
- 사용자별 최대 3연결, 15초 하트비트, 2시간 타임아웃
- 대시보드 데이터 30초 폴링 자동 갱신 (TanStack Query)

### 3.6 종목 검색 및 관심종목

- Cmd+K 단축키 종목 검색 (한글 초성 검색 지원)
- 종목 상세 페이지 (현재가, 캔들스틱 차트)
- 관심종목 추가/삭제 (KRX/NYSE/NASDAQ 마켓 구분)

### 3.7 알림 시스템

- 가격 알림 설정 (목표가 상한/하한)
- SSE 스트림에서 실시간 조건 체크 + 쿨다운(1h) + 자동 비활성화
- 인앱 알림 센터 (벨 아이콘 + 미읽음 배지 + 드롭다운 패널)
- 단건/전체 읽음 처리

### 3.8 KIS 계좌 관리

- 다중 KIS 계좌 등록/수정/삭제
- 자격증명 연결 테스트
- 계좌 유형 선택 (일반/ISA/연금저축/IRP/해외주식)
- 모의투자/실전투자 토글

## 6. 인프라 및 운영

### 6.1 배포

- Docker Compose 4서비스 (frontend, backend, postgres, redis) + backup 서비스
- 멀티 스테이지 빌드 (백엔드 2단계, 프론트엔드 3단계)
- 배포: `https://joonwon.dev` (프론트), `https://api.joonwon.dev` (백엔드)

### 6.2 CI/CD (GitHub Actions)

7개 워크플로우: backend CI, frontend CI, deploy, docker-build, CodeQL, E2E, dependabot-auto-merge. 경로 기반 트리거로 변경 없는 서비스 빌드를 건너뛴다.

### 6.3 DB 백업

- 일일 `pg_dump` (매일 02:00, custom format)
- 보관 정책: daily 7개 + weekly 4개 + monthly 3개
- Cloudflare R2 외부 스토리지 백업
- 복원 테스트 검증 완료 (2026-03-21)

### 6.4 모니터링

- Sentry: 백엔드 + 프론트엔드 에러 트래킹, APM
- UptimeRobot: 헬스체크 엔드포인트 5분 간격 모니터링
- structlog: X-Request-ID 기반 요청 추적, MetricsMiddleware로 응답시간 기록
- 데이터 무결성 헬스체크: price_snapshots 갭 감지, holdings 수량 정합성, 고아 레코드 감지

### 6.5 비용 최적화

- KIS API: `asyncio.gather()` 병렬 호출, 장외 시간 SSE 비활성화
- Redis: 토큰 캐시(24h), 현재가 캐시(5분), 종목 마스터(24h) — 총 ~510KB
- Docker: Alpine 기반 이미지, standalone 빌드, 멀티 스테이지

## 8. 평가 및 개선 방향

### 강점

- **실시간 데이터 파이프라인**: KIS API 비동기 병렬 호출 + Redis 캐시 폴백으로 장애 내성 확보
- **높은 테스트 커버리지**: 백엔드 93%로 안정성 보장
- **완전한 CI/CD**: lint → test → build → deploy 자동화
- **보안 계층**: JWT rotation, AES-256-GCM, 보안 헤더, 레이트 리밋 다중 방어
- **실제 매매 기능**: 단순 조회를 넘어 국내/해외 실매매까지 지원

### 개선이 필요한 영역

- **프론트엔드 테스트 커버리지 확대**: MSW 설정 + 주요 컴포넌트 테스트(16개) 완료되었으나, Visual regression 및 Load testing 미구현. 커버리지 수치 측정 및 확대 필요
- **분석 고도화**: 기간 필터, 벤치마크 비교 (KOSPI200/S&P500), 리스크 지표 (Sharpe ratio, MDD) 미구현
- **알림 확장**: 인앱 알림만 존재, 이메일/푸시 알림 미지원
- **운영 모니터링**: Docker 볼륨 디스크 사용량, TLS 인증서 갱신 체크 미구현
- **데이터 내보내기**: CSV만 지원, Excel(xlsx) 및 세금 계산기 미구현
- **매니지드 인프라 전환**: 단일 서버 의존성 — Neon(PostgreSQL) + Upstash(Redis) 전환 검토 중
